# Palette Agent Ansible Playbooks

Ansible playbooks for registering and deregistering edge hosts with [Spectro Cloud Palette](https://docs.spectrocloud.com/deployment-modes/agent-mode/install-agent-host/) in agent mode.

## Prerequisites

- Ansible installed on the target host
- A Palette API key with tenant-level `edgeToken.create` and project-level `edgehost` permissions
- Network access to `api.spectrocloud.com`

## File Structure

```
palette_agent/
├── ansible.cfg              # Fact caching config
├── inventory.ini            # Target hosts (defaults to localhost)
├── group_vars/all.yml       # Palette API key, endpoint, project name
├── deregister.yml           # Standalone deregister playbook
├── register.yml             # Standalone register playbook
├── site.yml                 # Combined: deregister then register
└── tasks/
    ├── deregister_tasks.yml # Deregister task list
    └── register_tasks.yml   # Register task list
```

## Configuration

Edit `group_vars/all.yml` with your Palette credentials:

```yaml
palette_api_key: "<your-api-key>"
palette_endpoint: "api.spectrocloud.com"
palette_project_name: "<your-project>"
```

## Usage

```bash
# Register this host with Palette
ansible-playbook register.yml --become

# Deregister this host from Palette
ansible-playbook deregister.yml --become

# Deregister (if registered) then re-register
ansible-playbook site.yml --become
```

## What Each Playbook Does

### register.yml

1. Resolves the project UID from the project name
2. Creates a registration token via the Palette API (tenant-scoped, 30-day expiry)
3. Installs agent dependencies and downloads the Palette agent install script
4. Writes the `user-data` configuration and runs the installer
5. Patches `/run/stylus/userdata` with the registration token and restarts the agent
6. Polls the Palette API until the host appears as registered
7. Saves project UID, token UID, and edge host UID to `/etc/ansible/facts.d/palette.fact`

### deregister.yml

1. Reads the device ID from `/run/stylus/userdata`
2. Finds the edge host in Palette by device ID
3. Deletes the edge host via the Palette API
4. Stops all `spectro-*` systemd services
5. Cleans up `/var/lib/spectro`, `/run/stylus/userdata`, and `/oem/.spectrocloud/authToken`

### site.yml

Runs deregister then register in sequence. Safe to run if the host is not currently registered (deregister will skip gracefully).

## Local Facts

Generated secrets and runtime identifiers are stored in `/etc/ansible/facts.d/palette.fact` (JSON):

```json
{
  "project_uid": "...",
  "project_name": "...",
  "registration_token_uid": "...",
  "edge_host_uid": "...",
  "edge_device_id": "..."
}

```

## Notes

- The Palette API has strict rate limits (10 req/sec on auth endpoints). All API calls include retry logic with backoff. If the agent gets stuck in a rate-limit loop after registration, stop it briefly (`systemctl stop spectro-stylus-agent`), wait 2 minutes, then restart.
- The agent reads its config from `/run/stylus/userdata` at runtime, not `/var/lib/spectro/userdata`. The register playbook patches both locations.
- Edge host device IDs are generated by the agent (e.g. `edge-<hash>`) and do not match the system hostname.
- Registration token creation is tenant-scoped and must not include a `ProjectUid` header. The `defaultProjectUid` is set in the request body instead.
